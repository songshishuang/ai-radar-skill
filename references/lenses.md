# 研报视角（lens）

同一批 AI 资讯，不同身份的人关心的点完全不同。lens 决定**重要度怎么加权**和**深度解读往哪个方向写**。默认 `pm`。

每个视角下，给定基础重要度后做 ±1~2 的调整——让真正切中该身份痛点的条目浮到必读区。

---

## pm（产品经理，默认）

**关心**：产研提效、商业产品机会、竞品动向、可落地的用户价值。

**加权**：
- 上浮：研发提效工具（编码 agent / 工作流自动化）、新产品形态与商业模式信号、大厂产品策略变化。
- 下沉：纯学术理论、底层训练 trick（除非直接降低产品成本）。

**深度解读重点**：
- 启示写「对产研流程能怎么用」——具体到某个环节、某个工具替换。
- 机会信号写「能切什么产品 / 什么付费场景 / 哪个长尾市场」。
- 行动写「本周能做的调研 / 试点 / 竞品对标」。

---

## engineer（工程师 / 技术负责人）

**关心**：能不能用、怎么集成、性能与成本、踩坑与最佳实践。

**加权**：
- 上浮：开源项目与权重发布、SDK/框架/工具链、推理优化、可复现的技术突破、benchmark。
- 下沉：融资/IPO/纯商业新闻（给一行即可）、市场营销向产品发布。

**深度解读重点**：
- 启示写「技术选型影响、集成路径、迁移成本」。
- 机会信号写「能替换现有哪套方案、省多少成本/延迟」。
- 行动写「本周跑通 demo / 起 benchmark / 评估接入」，给具体仓库或命令方向。

---

## investor（投资 / 战略）

**关心**：市场格局、资本流向、商业化拐点、护城河与竞争。

**加权**：
- 上浮：融资 / IPO / 并购 / 营收数据 / 大客户签约 / 市场份额变化、平台级范式转移。
- 下沉：单点技术细节（除非构成壁垒）、开发者工具的功能更新。

**深度解读重点**：
- 启示写「对赛道格局、估值锚、竞争态势的影响」。
- 机会信号写「哪个细分赛道被验证 / 出现窗口、谁受益谁承压」。
- 行动写「该盯的指标（ARR/NRR/份额）、该跟踪的招股书/财报、可对标标的」。

---

## researcher（研究 / 学术）

**关心**：方法创新、SOTA、可复现性、未解问题与研究方向。

**加权**：
- 上浮：论文（尤其 HF Daily Papers）、新架构/训练方法、评测与数据集、模型能力边界。
- 下沉：产品发布、商业新闻、工具更新（给一行）。

**深度解读重点**：
- 启示写「方法的新意、与现有工作的关系、局限」。
- 机会信号写「打开了什么研究方向、什么问题仍未解」。
- 行动写「值得精读的论文 / 可复现的实验 / 可跟进的 baseline」。

---

## 用法

1. 加工每条时，先给基础重要度，再按当前 lens 的加权规则调整。
2. 必读区的 so-what 和附录的「启示/机会/行动」三段，全部按当前 lens 的「深度解读重点」来写。
3. 视角只换**侧重与措辞**，不改**事实**——同一条资讯的客观内容对谁都一样。
