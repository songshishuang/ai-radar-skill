# 日报 → 京ME 卡片推送格式

把生成好的日报压成一张京ME 消息卡片，用 **`jdme-push`** skill 推送给 ERP（个人）或群号。本文件固化**卡片样式与正文结构**；发送凭证、内网、预览确认都在 `jdme-push` skill 里，两者不混（凭证绝不写进本仓）。

> 适配京ME；换企业微信 / 飞书 / 钉钉等其它 IM 卡片时，**正文结构（每条挂链接、分区、精简）可直接复用**，只换字段名与发送脚本。

## 卡片字段（params.json）

| 字段 | 固定值 / 规则 |
|---|---|
| `title` | `📡 AI 情报日报 · {YYYY-MM-DD}`（周报 `📡 AI 情报周报 · {YYYY-Www}`，月报类推）|
| `subtitle` | `{lens 中文}视角 · ai-radar`（pm→PM / engineer→工程 / investor→投资 / researcher→研究）|
| `theme` | `red`（日报醒目）；周报 `blue`、月报 `purple` |
| `banner` | 默认（AI News 配图，jdme-push 内置，不必传）|
| `button` | `{ "text": "看完整日报", "url": "{在线站对应 period 的 URL}" }` |
| `recipients` | **由用户指定**（ERP 含字母 / 群号纯数字），绝不写死、绝不脑补 |

## 正文 content（me_md）结构

**铁律**：
1. **每条都挂超链接** `[显示文字](url)`——必读、每个 GitHub 项目、每条 X 动态都要点得开（裸 URL 不保证变蓝）。
2. 卡片是「扫读版」，比网页日报更**精简**——必读取 Top 3-5、GitHub 取 4-6、X 取真有动态的；摘要压成半句。
3. **控制总长**（IM 里太长体验差），约 12-18 行为宜；想看全的点按钮去网页。
4. 信息少 / 某区无内容就**整区省略**，不硬凑（同 ai-radar 诚实原则）。

模板：

```
**⚡ 速览**：{一句话概括，可含 1 个关键链接}

**🔥 必读**
· [{标题}]({url}) —— {so-what 半句}
（Top 3-5，每条一行带链接）

**🔧 GitHub 高星开源（产品×技术）**   ← 工程/研究 lens 默认带；PM 当日有料 / 新闻少时带
· [{owner/repo}]({url}) ⭐{stars} · {一句话定位}
💡 趋势：{一句归纳}

**🐦 本周 X 动态**   ← 有动态才带；搜不到整区省略
· [{人物 + 动态}]({url})

完整版 → [AI 情报站 · {period}]({在线站 URL})
```

## 发送流程

1. 生成日报后，按上表把内容压成京ME 卡片版正文（**每条带链接**）。
2. 写 `params.json`（`recipients` 用用户当场指定的 ERP / 群号）。
3. 调 **`jdme-push`** skill：它负责预览 → 等用户确认 → 京东内网发送 → 回报每个收件人成功/失败。**绝不跳过确认、绝不假装成功**。

## 基准样例（2026-06-25 实发版）

- `title`：`📡 AI 情报日报 · 2026-06-25` · `subtitle`：`PM 视角 · ai-radar` · `theme`：`red`
- 正文 12 个超链接（速览 1 + 必读 1 + GitHub 6 + X 动态 3 + 完整版 1）
- `button`：`看完整日报` → 在线站 6-25
- 收件人：ERP `songshishuang`（用户指定）
